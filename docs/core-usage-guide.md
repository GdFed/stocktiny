# 项目核心功能运用指南

目标：项目启动后，基于内置的情绪周期、策略引擎、天梯、四维分析与 AI 分析模块，提供“从开盘前到收盘后”的高效操作打法，帮助你在不同市场阶段做出更稳健、更具结构化的交易决策。

参考资料：
- 接口说明与回退机制：src/hooks/api/README.md
- 系统结构与数据流分析：docs/analysis.md
- 本地 Mock 数据源：src/data/strategyData.ts

更新时间：2026-03-03

---

## 一、快速启动与环境准备

- 启动开发环境
  - 安装依赖：npm i
  - 启动服务：npm run dev（默认 http://localhost:5173）
  - 构建/预览：npm run build && npm run preview
  - 如端口冲突：npm run dev -- --port 5174

- 可选：接入真实后端（保持 Mock 兜底）
  - 在项目根目录创建 .env.local：
    ```
    VITE_API_BASE=https://api.example.com
    VITE_API_TIMEOUT=8000
    ```
  - 或在各 Hook 中将路径从 '/ai/analysis' 改为你的真实网关路径（如 '/api/ai/analysis'）。
  - 校验失败或网络异常会自动回退到本地 mock（isFallback=true），UI 仍可正常工作。

- Node 版本建议
  - 建议 Node.js 18+，以获得更好的 ES 模块与 Vite 兼容性。

## 二、界面模块与职责（按实战决策链路排序）

项目将交易决策流程拆解为若干板块，每个板块对应一个核心认知维度：

1) EmotionCycleDashboard（情绪周期仪表盘）
- 数据：emotionPhases
- 用途：识别市场所处“冰点→复苏→发酵→高潮→分歧→退潮”阶段，并给出仓位建议与操作纪律。
- 关键：阶段更迭时调整整体仓位与策略风格（由进攻转防守，或由防守转观望）。

2) LadderBoard（连板天梯）
- 数据：mockLadderData
- 用途：观察高度与梯队是否完整、谁是“空间龙/板块龙/总龙头”，确认主线与跟风质量。
- 关键：处于高位时谨慎后排追击，更偏向“强者恒强”的主线中军/龙头。

3) FourDimensionSystem（四维选股系统）
- 数据：mockFourDimensionData（theme / character / technical / fund，各自评分 + 总分）
- 用途：从“题材强度、个股性格、技术结构、资金验证”四维评估标的。
- 关键：总分与单项分双维筛选；例如 totalScore≥85 且 fund/technical≥80，更具持续性。

4) AIAnalysisCenter（AI 智能分析中心）
- 数据：mockAIAnalysis（带状态机与打字机效果）
- 用途：根据当前市场模拟给出目标标的与建议（emotionPhase、推荐题材、建议文案、置信度）。
- 操作：点击“重新分析”触发新版建议（当前使用 mock；接入真实 LLM 后可替换为 SSE/流式）。

5) StrategyEngine（策略引擎）
- 数据：buyStrategies / sellStrategies
- 用途：将“何时买”“如何卖”的规则清晰化，标注执行条件与仓位建议。
- 关键：只执行状态为 'executable/triggered' 的策略，避免随意化操作。

6) PositionManager（仓位管理）
- 依据情绪周期与策略级别确定总仓位与单票上限（如主线重仓、后排小仓/不做）。
- 与 RiskControl 联动：达到风控红线自动降仓或杜绝开新仓。

7) RiskControl（风险控制）
- 数据：riskRules
- 用途：全局风控红线（禁止无题材纯涨停、禁止亏损加仓、禁止单票>50% 等）。
- 关键：始终保持开启；若触发，优先执行“清仓/降仓/不新开仓”动作。

8) ReviewChecklist（复盘清单）
- 数据：checklistSteps
- 用途：收盘后 30 分钟内完成“市场扫描→梯队梳理→龙虎榜→次日预案”，沉淀下一日计划。
- 关键：不要跳步，严格记录并形成闭环。

## 三、标准化“日内三段式”实战流程

A. 开盘前（15 分钟）
- 步骤：
  1) EmotionCycleDashboard：快速判定所处阶段（对应仓位建议）。
  2) LadderBoard：更新连板高度与梯队完整度，识别总龙头与板块龙头。
  3) FourDimensionSystem：针对候选标的查看总分与资金验证（fund score/seatQuality）。
  4) RiskControl：确认红线开启状态（不要为个股或题材关闭红线）。
  5) StrategyEngine：列出“今日可能触发”的买入策略清单（尤其 Level 2 确认性买入）。

- 决策：
  - 冰点/退潮：观望为主；仅少量试错首板/不做。
  - 复苏/发酵：积极关注主线中军与 1→2、2→3 的机会。
  - 高潮：持有龙头，但不追后排，筹备撤退预案。

B. 盘中（30-60 分钟关键窗口）
- 步骤：
  1) AIAnalysisCenter：“重新分析”获取提示（当前为 mock 逻辑），结合 LadderBoard 动态微调目标。
  2) StrategyEngine：满足“确认性买入”的硬条件（回封瞬间/量能持续）→ 执行；否则不追。
  3) PositionManager：按阶段与策略级别分配仓位；严控单票≤50%（风险规则）。
  4) RiskControl：一字首开接力、跌停撬板、高位缩量加速等一律禁止。

- 决策：
  - 仅当“题材强度 + 技术结构 + 资金验证”三者同向时才加仓。
  - 盘中板块分歧时，去弱留强，立刻执行被动止盈或止损纪律。

C. 收盘后复盘（20-30 分钟）
- 步骤：
  1) ReviewChecklist：逐项完成（涨停/炸板/溢价率/天地板/梯队/龙虎榜）。
  2) FourDimensionSystem：回头验证今日主线与目标股四维分数与实际表现是否一致。
  3) EmotionCycleDashboard：评估是否进入“分歧/退潮”，为次日预案做准备。
  4) StrategyEngine：写入次日买入条件（价格区间、量能阈值）与卖出触发条件。

## 四、各模块操作要点与阈值提示

1) 情绪周期（EmotionCycleDashboard）
- 阶段与提示（源自 strategyData.ts）：
  - 冰点（limitUp<30，炸板率>40%）：空仓观望，只试错首板；总仓位<20%。
  - 复苏：试错新题材首板与 1→2；总仓位 20-40%。
  - 发酵（主线明确，梯队完整）：重仓主线龙头，重点做 1→2、2→3；总仓位 60-80%。
  - 高潮（满屏涨停，高度>7）：持有龙头，不做后排；准备撤退。
  - 分歧（龙头炸板/天地板）：只去弱留强，不开新仓；仓位<30%。
  - 退潮（A 杀/核按钮）：绝对空仓。
- 实操：阶段变化优先于个股信号；仓位与纪律先行，方向后到。

2) 连板天梯（LadderBoard）
- 看点：
  - 高度：空间龙高度与跟风梯队是否“654321/4321”完整。
  - 结构：中军是否启动；是否出现大面积补涨与后排脉冲。
- 建议：
  - 高潮期不追后排；发酵期优先中军/龙头；分歧期仅持强去弱。

3) 四维选股（FourDimensionSystem）
- 推荐阈值（可作为参考经验值）：
  - totalScore ≥ 85；fund ≥ 80；technical ≥ 80；theme 题材明确且 limitUpCount 高。
  - midCapStarted=true 时，中军启动代表持续性更优。
- 发现冲突（如技术分高但资金分弱）→ 降低胜率预期或放弃。

4) AI 智能分析（AIAnalysisCenter）
- 操作：
  - 点击“重新分析”按钮，2s 后给出建议文案（模拟打字机呈现）。
- 解读：
  - emotionPhase + recommendedThemes + targetStock（含 boards 与类型）
  - confidence 为置信度（建议≥80 时结合四维与天梯确认）
- 接入真实 LLM：
  - 后端代理 POST /api/ai/analyze，返回 AIAnalysis；前端以流式替代打字机。

5) 策略引擎（StrategyEngine）
- 买入（示例）：
  - Level 1 预判：发酵期早期；竞价高开 3-5%+量能→ 挂涨停预埋；仓位 20%。
  - Level 2 确认：龙头≥3 连板；分时突破且量能持续→ 回封瞬间打板；仓位 40%。
  - Level 3 套利：高潮期补涨；低位首板、流通小；小仓快进快出；仓位 20%。
- 卖出（示例）：
  - 主动止盈：高度进入前三、次日封单显著衰减、同题材出现跌停→ 集合竞价先卖 50%。
  - 被动止盈：开盘未封且跌破分时均线、30 分钟内未回封→ 反弹至均线清仓。
  - 止损纪律：打板当日未回封/次日低开>3%/连板中断收阴→ 次日离场；单笔亏损≤-7%。

6) 仓位管理（PositionManager）
- 全局：
  - 单票≤50%，禁止亏损加仓。
  - 仓位随阶段自适应（参见情绪周期建议）；分歧/退潮立刻降仓。
- 组合：
  - 中军/龙头为核心仓位，补涨/套利仅小仓。保持 2-4 只核心持仓，剩余为试错仓。

7) 风险控制（RiskControl）
- 必守红线（节选）：
  - 禁止无题材纯技术涨停
  - 禁止一字板首开接力（T 字板除外）
  - 禁止高位缩量加速（5 板后一字板）
  - 禁止跌停板撬板
  - 禁止单票>50%、禁止亏损加仓、禁止盘后情绪化交易
- 触发即执行降级动作（不与市场争辩）。

8) 复盘清单（ReviewChecklist）
- 四步共 30 分钟：
  1) 数据扫描：涨跌停、炸板率、昨日涨停溢价等
  2) 梯队梳理：标注题材，识别总龙/板块龙/空间龙
  3) 龙虎榜：3 连板以上席位、机构/顶级游资介入、识别散户对倒
  4) 次日预案：阶段判定、目标股与买入条件、持仓卖点触发条件

## 五、真实接口接入与回退机制要点

- 接入方式（任一）：
  - 在 .env(.local) 设置 VITE_API_BASE；保留 Hook 的相对路径（如 '/ai/analysis'）。
  - 直接将 Hook 路径改为 '/api/...'（与后端协商统一网关）。
- 校验策略（validate）：
  - 各 Hook 在获取数据后做类型校验；失败不抛错，改用本地 mock（isFallback=true）。
  - 若后端返回结构不同：先改 src/types/index.ts，再更新 Hook 内 validate。
- 手动重试：
  - 大多模块提供“刷新/重新分析”交互（或页面刷新）以触发 refetch。
- 故障降级：
  - 网络/超时/服务端错误 → ApiError → fallback；保证 UI 可用性与演示连续性。

详情见：src/hooks/api/README.md

## 六、常见市场场景“战术剧本”

1) 发酵期早期（主线初定）
- 行动：
  - LadderBoard：确认中军与总龙
  - FourDimension：fund/technical≥80，theme 明确
  - StrategyEngine：执行 Level 1/2（优先确认性买入）
  - PositionManager：总仓 60-80%，核心仓位给主线中军/龙头

2) 高潮期（全面狂热）
- 行动：
  - 持有龙头，不做后排
  - 准备撤退：封单/高度/分歧信号一旦出现，主动止盈
  - 风控红线全部开启（特别是首开接力、缩量加速）

3) 分歧期（龙头炸板/天地板频现）
- 行动：
  - 去弱留强，谨慎新开仓
  - 降仓至<30%，重点保住既有收益
  - FourDimension：优先保留分数高且资金验证强的持仓

4) 退潮期（核按钮/普遍 A 杀）
- 行动：
  - 绝对空仓；复盘为主，等待冰点
  - 风控红线+纪律优先；不做“抄底刀口”

## 七、常见问题与排错

- 无法启动/白屏
  - npm i 是否成功；Node 版本是否 18+；控制台是否有构建报错
- 接口 404/超时
  - 检查 VITE_API_BASE 与路径是否一致；超时可调 VITE_API_TIMEOUT
- 数据校验失败（isFallback=true）
  - 按 src/hooks/api/README.md 更新类型与 validate；或先使用 mock 继续联调
- UI 刷新节奏
  - 开盘前 1 次、盘中关键窗口按需 2-3 次、收盘后 1 次；避免频繁抖动造成误判

## 九、快捷清单（Checklist）

- 每日三段式：
  - 开盘前：情绪周期→天梯→四维→风控→策略清单
  - 盘中：AI 提示→确认性买入→仓位控制→风控执行
  - 收盘后：复盘四步→次日预案
- 永久红线：
  - 无题材纯涨停不做；一字首开接力不做；高位缩量加速不做；跌停撬板不做；
  - 单票不超 50%；禁止亏损加仓；禁止盘后情绪化交易
- 执行纪律：
  - 只做“可验证的强势”：题材明确 + 技术结构 + 资金验证三者共振
  - 计划交易，按触发条件执行；不计划的交易一律不做
